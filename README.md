# Reflexión sobre el despliegue de un sitio estático con GitHub Pages
En esta práctica he realizado el despliegue de un sitio web estático utilizando Git y GitHub Pages.

## Control de versiones con Git
El uso de Git ha permitido llevar un seguimiento ordenado y preciso de la evolución del proyecto. Cada cambio relevante se ha registrado mediante commits.
En un principio no ha sido complicado

## Uso de GitHub
La vinculación del repositorio local con GitHub ha permitido almacenar el proyecto de forma segura en la nube.
Yo muchas veces lo he utilizado para subir actividades de asignaturas por si habia algun problema con la maquina virtual o con el disco duro. Asi no pierdo las actividades en caso de que las necesite
<img width="1917" height="1079" alt="Github" src="https://github.com/user-attachments/assets/ad69dab8-3fa3-411a-9c72-ca2db1880f1d" />

## Publicación automática con GitHub Pages
La utilización de GitHub Pages ha facilitado el despliegue del sitio web sin necesidad de configurar un servidor propio.
Yo no la he vinculado a ningun dominio porque no tengo. Pero esto es suficiente para entenderlo por ejemplo cada vez que se realiza un `git push` a la rama principal, el sitio se actualiza automáticamente

## Conclusión
Esta práctica me ha permitido comprender la importancia del control de versiones, la documentación y el despliegue automatizado en el desarrollo web.
Herramientas como Git y GitHub Pages no solo facilitan el trabajo individual, sino que son esenciales en entornos profesionales donde la colaboración grupal
